# 终极卡顿修复与第一阶段总结计划

您反馈的“视频中间依然存在卡顿”非常关键。这说明之前的优化虽然解决了大部分问题，但核心的**帧同步**（Frame Sync）问题依然存在。

## 问题根源分析 (Root Cause Analysis)
1.  **浏览器解码延迟**: 即使提前 `play()`，视频解码器（Decoder）在首帧渲染时仍可能阻塞主线程，导致掉帧。
2.  **透明度动画性能**: CSS `opacity` 动画虽然有 GPU 加速，但涉及图层合成（Compositing），在大尺寸视频（1080p/4K）上依然可能引起重绘（Repaint）。
3.  **计时器精度**: `timeupdate` 事件触发频率不稳定（通常 250ms 一次），可能导致错过最佳切换点。

## 解决方案：双缓冲 + 强制关键帧预热 (Double Buffering & Keyframe Warm-up)

我们尝试一种更底层的解法：**不依赖 CSS 动画，而是使用 JS 强制同步切换**。

### 1. 修复视频卡顿 (`src/components/HeroBackground.jsx`)
*   **策略调整**:
    *   **移除 CSS Transition**: 放弃 CSS 的平滑过渡，改用 JS `requestAnimationFrame` 手动控制透明度。这样能精确到每一帧。
    *   **强制预热**: 在备用视频 `play()` 后，不要立即开始淡入。等待 `onPlaying` 事件触发，确保浏览器真的开始输出了，再开始淡入。
    *   **减少重叠**: 缩短 Cross-fade 时间（从 1.5s 减至 0.8s），减少双视频同时解码的时间窗口，降低 CPU/GPU 压力。
