# 视频无缝循环优化计划

您指出的问题非常关键。由于素材本身（5秒）并非为无缝循环设计的（首尾帧不一致），直接的 `loop` 播放会导致明显的“跳变”或“卡顿”感。

## 解决方案分析

### 方案 A：来回播放 (Ping-Pong Loop) —— **推荐**
*   **原理**: 当视频播放到结尾时，不直接跳回开头，而是**倒序播放**回开头。这样 `A -> B -> A` 的过程在视觉上是绝对连续的。
*   **实现**: 不需要修改视频文件。我们可以通过 JS 监听 `timeupdate` 事件，或者更简单地，使用两个 `<video>` 叠加，通过透明度交叉淡入淡出（Cross-fade），但这比较消耗性能。最轻量的方法是**使用 CSS 动画或 JS 控制 playbackRate**，但 JS 控制倒放可能在某些浏览器上不流畅。
*   **优化版 Ping-Pong**: 直接在前端用 JS 控制：播放到 4.5s 时，将 `playbackRate` 设为 -1（倒放），播放到 0.5s 时设为 1（正放）。这需要浏览器支持负倍速（大部分现代浏览器支持，但有兼容性风险）。

### 方案 B：双层视频交叉淡入淡出 (Cross-fade)
*   **原理**: 在视频快结束（比如 4s）时，让另一个从头开始播放的视频层逐渐显现（Opacity 0 -> 1），覆盖在当前视频上。
*   **优点**: 视觉极其平滑，是电影工业级的处理手法。
*   **缺点**: 需要同时渲染两个 Video 标签，但对于短视频来说性能损耗可控。

### 方案 C：使用 CSS 模糊遮罩 (Blur Mask)
*   **原理**: 在循环点（5s -> 0s）的一瞬间，让画面快速变黑或变模糊，掩盖跳变。
*   **缺点**: 会有明显的“闪烁”感，不推荐用于背景。

## 选定方案：方案 B (Cross-fade Loop)
为了达到最完美的视觉效果，我建议采用 **双层视频交叉淡入淡出** 的策略。虽然代码稍复杂，但效果最好。

## 实施步骤 (`src/components/HeroBackground.jsx`)

1.  **状态管理**: 使用 React `useState` 维护两个视频层的状态（`active` / `inactive`）。
2.  **双 Video 结构**: 渲染两个绝对定位重叠的 `<video>`。
3.  **定时逻辑**:
    *   主视频播放到 `duration - transitionTime` (比如 4s) 时，启动副视频播放。
    *   副视频开始淡入 (`opacity: 0 -> 1`)。
    *   主视频淡出或保持在底部。
    *   当副视频完全接管后，重置主视频到 0s 并等待下一次轮替。
    *   *简化版*: 其实对于背景视频，更简单的做法是：当视频 A 播放到 4s 时，视频 B (opacity:0) 开始播放并渐变到 opacity:1。视频 A 在 B 完全显示后，跳回 0s 并变透明，等待下一次。

**修正**: 考虑到实现复杂度与性能，还有一个**更取巧的方案 D**：
**镜像拼接 (CSS Mirror)**: 既然是星球自转，如果左右边缘差异不大，其实看不出跳变。但如果差异大，Ping-Pong（方案A）是成本最低且效果不错的。

**最终决策**: 让我们先尝试 **方案 A 的变体 (JS Ping-Pong)**。利用 `requestAnimationFrame` 监控进度，如果接近尾声就倒放。如果效果不好（倒放不流畅），我们再切换到 **方案 B (Cross-fade)**。

**补充**: 考虑到兼容性，最稳妥的其实是 **方案 B (Cross-fade)**。我将为您实现一个轻量级的 Cross-fade 逻辑。
