## ✨ 叙事主题设计思路

整个网页以「小行星」为核心叙事意象，构建一段完整的宇宙旅程：

| 页面 | 主题 | 叙事含义 |
|------|------|----------|
| 第一页 | 行星诞生 | 我是谁——初次相遇 |
| 第二页 | 星光积攒 | 我从哪里来——教育积累 |
| 第三页 | 星轨初具 | 我走过哪些路——项目经历 |
| 第四页 | 星辰涌现 | 我创造了什么——实践成果 |
| 第五页 | 星轨归处 | 再次相遇——感谢相逢 |

视觉上以粉紫蓝星空为底，星球、光球、星轨弧线贯穿始终，让面试官在浏览的过程中感受到一段完整的「宇宙旅程」，而不只是在翻阅一份简历。

## 📖 各页面设计说明

### 第一页 · 行星诞生
打字机渐显动效逐行呈现自我介绍文案，文字以双层text-shadow（微弱深色投影+白色光晕）在复杂星空背景上保持清晰可读。两个交互按钮——「星图下载」触发简历PDF下载，「星轨相接」弹出联系方式弹窗。

### 第二页 · 星光积攒
以一条自下而上逐渐发光的星轨线为视觉核心，标注安徽与上海两个节点，点击节点弹出Glassmorphism毛玻璃卡片，展示对应的教育经历。

### 第三页 · 星轨初具
以个人为中心向外扩散的同心圆轨道，承载百度、美团、Lovart三段工作经历。hover轨道高亮，点击弹出半屏侧拉卡片展示详细项目内容。

### 第四页 · 星辰涌现
三个带行星环的发光球体，初始聚拢，hover后以弹性动画散开。点击进入左侧条目列表+右侧阅读区的双栏阅读界面，展示Vibe Coding记录、产品分析文档、AI学习理解三类成果内容。

### 第五页 · 星轨归处
完全复用第一页的视觉结构与动效，替换为收尾文案，与开篇形成叙事闭环。「星图下载」与「星轨相接」再次出现，赋予更深的叙事含义。

## 🤖 AI工具使用方法论

这个项目从设计到实现，全程借助AI工具完成，以下是我的使用方式：

**1. 用Claude做架构决策**
在每一个设计节点，我先和Claude讨论方向与逻辑——页面的叙事结构应该怎么组织？某个交互形态是否合理？视觉问题的根本原因是什么？Claude在这里扮演的是产品顾问和设计评审的角色，输出的是思路和决策，而非只是代码。

**2. 用Trae做代码实现**
设计方向确认后，我在Trae IDE中与AI协作落地——将经过思考的需求转化为精准的提示词，和Trae的Builder逐步调试实现。提示词越具体（包含数值、颜色值、动效参数、边界情况），实现质量越高。

**3. 迭代而非一次成型**
没有任何一个页面是一次提示词就完成的。每次实现后我都会评估效果，识别问题根源，再给出针对性的修改指令。这个过程本身就是产品迭代思维的体现。

**4. 核心认知**
AI工具改变的不是「做什么」，而是「怎么做」。产品经理的价值在于清晰定义问题、做出判断、把控质量——这些在AI时代依然不可替代。
